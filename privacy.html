Privacy Policy

Last updated: May 2026

PrivateVault ("the app") is designed as a secure offline-first encrypted vault application. Your privacy and data security are fundamental priorities of the app.

1. Data Collection

PrivateVault does NOT intentionally collect, transmit, upload, sell, rent, or share your personal files or vault contents.

The app operates locally on your device and is designed without mandatory cloud synchronization or analytics services.

PrivateVault does NOT collect:

  • photos
  • videos
  • documents
  • vault passwords
  • clipboard history
  • contacts
  • messages
  • location data
  • microphone recordings
  • camera recordings
  • advertising identifiers
  • usage analytics
  • crash analytics
  • behavior tracking

2. Permissions Used

PrivateVault only requests permissions necessary for core vault functionality.

Storage & Media Permissions

  • READ_MEDIA_IMAGES
  • READ_MEDIA_VIDEO
  • READ_MEDIA_AUDIO
  • Document access via Android system file picker APIs

These permissions are used solely for:

  • importing user-selected files
  • opening encrypted content
  • exporting user-selected files
  • creating previews and thumbnails locally

PrivateVault does NOT request INTERNET permission for core vault operation.

3. Encryption & Security

PrivateVault uses local encryption technologies designed to protect user data stored inside the vault.

  • AES-256 encryption
  • Encrypted local storage
  • Password-protected vault access
  • Android Keystore integration where supported
  • Screenshot blocking on sensitive screens
  • Secure temporary file handling
  • No mandatory cloud dependency

Vault passwords are not uploaded to external servers.

4. Offline-First Operation

PrivateVault is designed to function primarily offline.

Core features including:

  • file encryption
  • vault browsing
  • document viewing
  • media playback
  • local search
  • thumbnail generation

operate locally on-device.

5. File Sharing

PrivateVault may allow users to export or share decrypted files ONLY when explicitly initiated by the user.

Users remain responsible for handling exported files securely outside the app.

6. Children’s Privacy

PrivateVault is not intentionally directed toward children under 13.

The app does not knowingly collect personal information from children.

7. Third-Party Services

PrivateVault does not intentionally integrate advertising SDKs or behavioral tracking SDKs in its core offline configuration.

If future optional online features are introduced, this Privacy Policy may be updated accordingly.

8. Google Play Data Safety

According to the current intended offline-first design:

  • No user data is sold
  • No personal data is shared with third parties
  • No mandatory analytics collection is enabled
  • No advertising SDKs are intentionally included
  • User files remain locally stored unless explicitly exported by the user

9. Security Disclaimer

While strong security measures are implemented, no software can guarantee absolute security in every possible environment.

Users are encouraged to:

  • use strong passwords
  • keep devices updated
  • avoid sharing decrypted exports unnecessarily
  • enable device lock security

10. Changes To This Policy

This Privacy Policy may be updated periodically to reflect:

  • feature updates
  • security improvements
  • Google Play policy requirements
  • legal compliance updates
