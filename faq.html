Frequently Asked Questions

Updated FAQ for PrivateVault’s latest offline vault, encryption, document editor, media viewer, secure sharing, performance, and privacy features.

Privacy & Offline Operation

Offline-first

Does PrivateVault require internet access?

No. PrivateVault is designed as an offline-first encrypted vault. Core vault features are intended to work locally on your device without internet access.

No cloud by default

Does PrivateVault upload my files to cloud storage?

No. Vault files remain local unless you manually export or share them outside the app.

Local processing

Are thumbnails, search, document previews, and OCR processed locally?

Yes. The app is designed to process thumbnails, indexing, previews, scanning, and organization locally without sending your vault content to external servers.

Permissions

Minimal permissions

Which permissions does the app use?

PrivateVault uses only permissions required for selected features, such as reading user-selected images, videos, audio, documents, and optional camera capture when you use the camera feature.

Camera

When is camera permission requested?

Camera permission is requested only when you use direct camera capture or document scanning features.

No internet core

Does the vault need INTERNET permission?

The core offline vault does not require internet permission. This helps reduce data-leak risk.

Encryption & Vault Security

Encryption

How are files protected?

Files are stored in encrypted form inside the vault. The app is designed around AES-256 encryption, secure local storage, integrity checks, and protected vault sessions.

Password

Does the app support long vault passwords?

Yes. The vault design supports strong passwords up to 127 characters.

Screenshots

Are screenshots blocked?

Yes. Screenshot and screen-recording protection is enabled for sensitive vault screens in the secure production build.

Integrity

How does PrivateVault reduce corruption risk?

The app includes layered integrity validation, transactional writes, metadata recovery, chunk checks, restore validation, and corruption-resistant encrypted file handling.

File Import, Export & Sharing

Import

Can I import many file types?

Yes. PrivateVault supports importing common media, documents, code files, archives, and other user-selected files into the encrypted vault.

Export

Can I export decrypted files?

Yes. Files can be exported in decrypted form only when you explicitly choose to export them to a selected location.

Share toggle

Can file sharing be disabled?

Yes. Settings include a toggle to enable or disable external file sharing. When disabled, decrypted file sharing outside the app is blocked.

Safe sharing

How does secure sharing work?

Shared files are prepared as temporary decrypted copies, passed to the selected app, and cleaned up after the sharing session.

Documents, Office Files & Code Editing

Office-style editor

Can I edit Word, Excel, PowerPoint, and PDF-related files?

The app includes a compact Office-style editing architecture with formatting toolbars, spreadsheet grid support, presentation layout support, document zooming, search, and conversion/export workflows.

Formatting

What formatting tools are available?

Formatting support includes bold, italic, underline, strike, font sizing, color, alignment, lists, tables, cells, media embeds, links, and document presets.

Find

Can I search inside opened documents or code files?

Yes. A Find button is available for text-based documents and code files in both viewer and editor modes. It is not shown for image or video files.

Code files

Can I open and edit coding files?

Yes. The app supports editing common code formats such as JavaScript, TypeScript, C, C++, Java, PHP, Python, Kotlin, Swift, HTML, CSS, JSON, XML, YAML, and more. APK files are inspectable but not executable.

Conversion, Save As & Protected Files

Convert

Can documents be converted to other formats?

Yes. PrivateVault includes a unified Save As and Export system for converting supported documents, spreadsheets, presentations, PDFs, images, and text formats.

Legacy formats

Does the app support old document formats?

The compatibility system includes support pathways for modern and older formats such as DOC, DOCX, XLS, XLSX, PPT, PPTX, RTF, TXT, CSV, ODT, ODS, ODP, PDF, and image formats.

Password files

Can I open password-protected files?

Yes. Supported password-protected documents and archives can be opened after entering their password. Password handling includes retry limiting, session isolation, and temporary buffer cleanup.

Save As

Can I save documents in newer or older formats?

Yes. The Save As system is designed to offer compatible output formats, including modern and legacy document formats where supported.

Media, Photo Editor & Video Player

Images

Can I view and edit images?

Yes. The app supports image viewing, zooming, panning, resizing, saving as a new image, filters, and basic image editing tools.

Video

Can videos play in portrait and landscape?

Yes. The video player supports dynamic rotation according to device rotation without forcing horizontal orientation.

Zoom

Does media support double-tap and pinch zoom?

Yes. Images and videos support double-tap zoom and pinch zoom with premium edge resistance and controlled panning.

Playback

Are video controls optimized?

Yes. The player includes precise scrubbing, auto-hiding controls, rotation-aware layouts, responsive gestures, and smoother seek behavior.

Camera Scan, OCR & Archives

Scanner

Can I scan documents with the camera?

Yes. The app includes camera document scanning with edge detection, perspective correction, OCR isolation, and multi-format export support.

OCR

Is OCR handled securely?

OCR is designed to run locally with sandboxing, input sanitization, language pack validation, and memory controls.

Archives

Can I create or unzip compressed files?

Yes. PrivateVault supports archive creation and extraction workflows, including encrypted archive handling, integrity verification, and protection against archive-bomb and path-traversal attacks.

Clipboard & Data Control

Clipboard toggle

Can I restrict copied vault data to stay inside the app?

Yes. Settings allow clipboard behavior to be internal-only or system-allowed, depending on your chosen policy.

Auto-clear

Does clipboard data expire?

Yes. Clipboard content can be auto-cleared after a timeout or when sensitive sessions are backgrounded.

Performance & Device Optimization

Low-end devices

Is PrivateVault optimized for budget phones?

Yes. The app includes low-end mode, preview downscaling, lightweight transitions, reduced effects, memory trimming, and background workload limits.

Mid-range devices

Does the app provide flagship-like smoothness on mid-range devices?

Yes. Mid-range optimization includes touch prediction, render-cache fusion, GPU burst control, frame stability boosting, and thermal-aware scaling.

High-end devices

Are high-refresh displays supported?

Yes. High-end optimization includes 90Hz/120Hz+ rendering paths, low-latency input, advanced animations, texture residency, and multicore workload distribution.

Battery

Does the app optimize battery usage?

Yes. Battery-aware systems reduce background work, animation intensity, rendering load, and heavy encryption tasks when needed.

Security Hardening

Sandboxing

How are risky files handled?

High-risk operations such as PDF parsing, archive extraction, OCR, thumbnails, font rendering, and media decoding are designed with sandboxing and capability restrictions.

Macros

Are document macros allowed?

No. Macro execution and active document content are blocked to reduce malicious document risk.

APK files

Can APK files be run inside the app?

No. APK files can be inspected in read-only mode, but they cannot be executed or installed by the vault.

Tamper defense

Does the app include runtime hardening?

The architecture includes anti-tamper awareness, debugger detection, hook detection, emulator heuristics, screenshot blocking, secure session cleanup, and hardware-backed key enforcement where supported.

Note: Some advanced document conversion, editing, OCR, sandboxing, and hardware-backed security behavior may depend on device capabilities, Android version, supported file type, and build configuration.
